distributed version control

08 Oct 2009, 01:13

My team has been using Mercurial as our version control system, and it is definitely pretty cool. We have run into some interesting “problems” though. Since the main Thunderbird repository is subversion, and we don’t have commit access, we have to submit patches that are reviewed and then committed. The real problem comes with branching, merging, and committing. Long story made short (or the moral of the story) is that newbies shouldn’t push/pull Mercurial repos from each other.
